tensorflow 学习教程（一）

内容概要

本篇学习教程主要针对tensorflow2.0,同时兼顾tensorflow1.0的部分代码特点进行总结。
今天主要学习了tensorflow中关于layer的定义，包括函数式/功能式API的概念及使用，子类API的概念及使用,以及Model与Layer的区别。下面就让我们逐一对这些内容进行了解吧！

函数式/功能式API

初识函数式API

函数式API（function API） 有时又称作功能式API，在这种方式下，我们写好的model的layer就像一个被写好的普通函数一样可以通过向函数传参（该层所要处理的数据）的方式被直接调用，增加了程序编写的灵活性。

一个全连接网络可能包含以下几个部分：输入层、隐层、输出层，有时还需要层间进行一个concatenate，我们现在就以这几种layer为例，看一下怎么以函数式API的方式进行编写。

  • 输入层
  • 隐藏层
  • concatenate层
  • 输出层
  • 将上述几层连接起来形成一个model
  • 查看model
  • 同时，所有的模型也能像层一样被调用*

    实现模型的多输入与多输出

  1. 函数式API使得多输入与多输出的应用变得简单，而Sequential API难以做到。
    主要就是通过上述构建函数式API的方法，使用keras.models.Model(inputs = [],outputs = []),inputs和outputs参数中传入列表的方式表示多输入与多输出，layers仍然使用上述函数式API方式定义
  2. 由于是多输出，可以针对不同的输出构造不同的损失函数，并对不同的损失函数赋予不同的权重以改变其对整体损失函数的影响
  3. 使用model.fit对模型进行训练时，训练集中的x和y是通过字典元组进行传参的

    子类API

    初识子类API

    keras的一个中心抽象是Layer层，层封装了状态（层的权重，对应__init__()函数定义）和从输入到输出的转换（调用，即层的前向传递，对应call()函数定义）

    权重的定义

  • 通过初始化函数定义weight
  • 通过tf.add_weight()函数定义weight
  • 将权重构建推迟到得知输入的形状之后：定义self.bulid()函数

    前向过程的定义

  • 在call（）函数中通过self.add_loss(value)实现损失张量的定义，而且这些损失（包括内部层创建的损失）可以由layer.losses()取回 这些loss可以在model.fit()函数中自动求和并添加到主损失中
  • 在call()函数中通过self.add_metric()方法添加度量指标，同add_loss（），同样可以通过layer.metrics访问获取，也可以通过fit()跟踪
  • 可以通过在子类API中添加get_config()方法将自定义层作为函数式模型的一部分进行序列化
  • 另外，基础 Layer 类的 init() 方法会接受一些关键字参数，尤其是 name 和 dtype。最好将这些参数传递给 init() 中的父类，并将其包含在层配置中：
  • 关于call()方法中的training特权参数
    这个参数的目的就是为了区分目前是处于训练阶段(training == True)还是推断阶段(training == False),因为某些layer（如BatchNormalization层和Dropout层）在训练时和推断时有不同的行为

Model和Layer

Model类和Layer类具有相同的API，但是Model类具有以下的特殊功能:

  • 公开内置训练、评估和预测循环：model.fit(),model.evaluate(),model.predict()
  • 可以通过model.layers获取其内部的层结构；
  • 公开保存和序列化API：save()以及save_weights()
